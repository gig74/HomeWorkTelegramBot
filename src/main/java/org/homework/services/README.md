
Добро пожаловать в директорию, содержащую классы, которые относятся к слою сервисов нашего Telegram бота.


## Краткое Введение

Сервисный слой приложения - это прослойка между пользовательским интерфейсом (в нашем случае, чат с ботом) и доступом к данным или внешним API. Здесь происходит организация основной бизнес-логики, обработка данных, полученных от клиента, а также подготовка данных для отправки или дальнейшей обработки.

## Классы Сервисов

### BasicHttpService

`BasicHttpService` является реализацией интерфейса `HttpService`, предоставляющая основные методы для отправки HTTP-запросов.

#### Основные методы:
- `sendGetRequest`: отправка GET-запроса и получение ответа в виде `String`.
- `sendPostRequest`: отправка POST-запроса с телом в виде `String` и получение ответа.
- `sendPutRequest`: отправка PUT-запроса, аналогично POST-запросу.
- `sendDeleteRequest`: отправка DELETE-запроса для удаления ресурсов.

### CommandServiceImpl

`CommandServiceImpl` служит реализацией интерфейса `CommandService`, содержащего методы для выполнения команд, приходящих от пользователя через бота.

### DataServiceImpl

`DataServiceImpl` реализует интерфейс `DataService` и отвечает за получение данных для ответа бота.

## Добавление нового сервиса

Для добавления нового сервиса в проект выполните следующие шаги:

1. Создайте новый класс в пакете `org.homework.services`.
2. Если ваш сервис должен реализовать специфичное API, следует сначала определить новый интерфейс в директории `api`.
3. Добавьте необходимую реализацию методов, следуя принципам SOLID и Clean Code.
4. Аннотируйте класс аннотацией `@Register` для автоматической регистрации в рамках системы DI (dependency injection).
5. Инжектируйте зависимости, используя аннотацию `@Resolve`.
6. Учтите логирование важных событий в вашем сервисе для упрощения отладки и поддержки.

Каждый сервис должен предоставлять четко определенные функции и быть максимально независимым от других компонентов системы для обеспечения гибкости и удобства в дальнейшем масштабировании и поддержке проекта.
